# 个人作品集网站开发过程总结

## 1. 项目目标

本次开发目标是基于个人简历与实施方案，从零搭建一个个人作品集网站。网站身份定位为：

- 精益工程师
- IE 工程师
- 工业工程师
- 制造业现场改善与质量管理方向

整体风格要求为稳重、制造业属性明确、适合 PC 端展示，版心控制在约 1700px，避免普通模板网站感。

## 2. 技术栈

项目使用 React + Vite 实现，当前主要技术栈如下：

- React 19
- Vite 6
- lucide-react 图标库
- 原生 CSS
- Intersection Observer 滚动显现动画
- CSS 动态颗粒渐变背景

开发与验证命令：

```bash
npm run dev
npm run build
```

本地预览地址：

```text
http://127.0.0.1:5173/
```

1. 首页 Hero
2. 个人经历
3. 精选项目

5. 底部联系方式

## 6. React Bits Grainient 背景尝试与白屏问题

开发过程中尝试接入 React Bits 的 `Grainient` 组件，用于网页头图下方背景。

该组件依赖：

```text
ogl
```

并使用 WebGL2 渲染动态渐变颗粒效果。

### 白屏原因判断

接入后页面出现白屏。服务端口仍然正常，`http://127.0.0.1:5173/` 返回 200，说明不是 Vite 服务挂掉，而是浏览器端页面渲染中断。

主要原因判断为：

- `Grainient` 依赖 WebGL2 和运行时 canvas 初始化
- 当前内置浏览器环境对该 WebGL 背景不稳定
- WebGL 初始化或渲染异常会导致 React 页面脚本中断
- 因此页面地址能打开，但 DOM 没有正常挂载，表现为空白页

这不是普通 CSS 样式冲突，而是动态 WebGL 组件在当前预览环境里的兼容性和初始化问题。

### 修复方式

为保证预览稳定，最终移除了 WebGL 组件渲染路径，并卸载 `ogl` 依赖。

同时保留视觉需求，改为纯 CSS 实现类似的颗粒渐变背景：

- 背景使用多层 `radial-gradient`
- 叠加细网格
- 叠加微颗粒纹理
- 加入轻微漂移动画

这样既保留了 React Bits 风格方向，又避免 WebGL 导致白屏。

## 9. 后续可继续优化方向

后续可以继续优化：

- 首页是否保留证件照，或改成纯图形/工业视觉块
- 项目卡片是否换成真实项目截图
- 增加移动端适配
- 增加项目详情弹窗或详情页
- 加入导航当前模块高亮
- 将联系方式按钮做成更明确的下载简历/发送邮件入口
- 如果仍希望使用 React Bits 原版 Grainient，可单独做兼容测试后再接入

本次个人站已经完成从基础搭建到视觉迭代的主要工作，当前版本定位清晰，内容围绕精益、IE、工业工程和制造现场改善展开。白屏问题来自 WebGL 动态背景组件的运行兼容性，而非普通页面结构冲突；目前已通过 CSS 版本背景完成稳定替代。
