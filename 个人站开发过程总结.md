# 汪金个人站开发过程总结

## 项目定位

本次个人站从简历内容和实施方案出发，搭建为一个面向制造业现场改善的个人作品集网站。网站身份定位为：

- 精益工程师
- IE 工程师
- 制造现场改善与质量管理方向

整体视觉目标是稳重、专业、有工业科技感，避免普通模板站的感觉。页面以深蓝、钢灰、白色为主色，少量金色作为重点强调色，适合 PC 端展示。

## 技术栈

项目采用 React + Vite 实现，当前主要技术包括：

- React 19
- Vite 6
- GSAP + ScrollTrigger
- lucide-react 图标
- 原生 CSS 动效与响应式布局
- WebP 图片资源优化

常用命令：

```bash
npm install
npm run dev
npm run build
```

本地预览地址：

```text
http://127.0.0.1:5173/
```

## 页面结构

当前网站为单页滚动式作品集，包含以下模块：

1. Hero 首页
2. Professional Experience 个人经历
3. Selected Projects 精选项目
4. Capabilities 个人优势
5. Contact 联系方式

## Hero 首页

首页经过多轮排版调整，最终采用高端作品集封面式布局。

当前保留的核心内容：

- 英文主标题：`PROJECTS`
- 姓名：`汪金`
- 标语：`从产线到数据，让制造更精益`
- 顶部导航栏

首页弱化了普通简历式信息，删除了过多身份说明和照片元素，让视觉中心更集中。整体参考创意机构与设计师作品集的首屏结构，使用居中导航、留白、英文大标题和缓慢进场动画，形成更强的第一印象。

## 个人经历页

个人经历页最初包含左侧头像信息区和右侧履历区，后续根据页面视觉统一性进行了精简。

当前结构重点保留右侧内容，并做成卡片化信息系统：

- Education
- Experience
- Toolkits
- Methods

卡片采用深蓝工业科技风样式，带有细边框、柔和背景层次和边缘光效。教育与经历按时间线信息重新排版，工具与方法保留最关键内容，避免堆砌。

## 精选项目页

精选项目页从 2x2 项目卡片逐步调整为纵向居中排列，更适合展示大图和项目成果。

当前项目包括：

- 铁芯质量提升项目
- 匝数错误防控体系
- K1 产线流程写实与效率提升
- 高压绕线工段培训体系搭建

项目图片使用工业风格 WebP 资源，并增加图片 reveal、轻微 parallax 和卡片进场动效。项目页删除了多余中文介绍，只保留英文标题 `Selected Projects`，让页面更干净。

## 个人优势页

个人优势页保留四个核心能力方向：

- IE / 精益改善
- 质量管理
- 工艺与现场认知
- 数据与工具应用

视觉上改为与项目卡片一致的动效语言，卡片具备边缘光效、悬停反馈和分批进场效果，使页面之间的体验更统一。

## 联系页

联系页作为整屏收尾页，当前文案调整为：

```text
期待与企业一起，让改善落地见效
```

页面保留电话、邮箱、地区和发送邮件入口。布局经过调整后整体向下移动，减少页面上方拥挤感，并让左下区域更平衡。

## 动效系统

后期引入 GSAP + ScrollTrigger，目标是让页面具备高端设计师作品集或创意机构官网的动效质感。

已实现效果：

- 首屏 opening animation
- Hero 标题遮罩揭开与位移归位
- 滚动到模块时英文大标题大幅进场
- 卡片 stagger 依次出现
- 项目图片 reveal 与轻微 parallax
- 卡片边缘 hover glow 光效
- reduced motion 兼容

动效节奏控制为慢速、丝滑、不过度打扰阅读。

## 性能优化

开发过程中重点审查了首屏加载、图片体积、WebGL、滚动监听和 GSAP 动画。

主要优化结果：

- 未使用视频背景，避免首屏资源过重。
- 移除 WebGL 动态背景，避免兼容性导致白屏。
- 使用 CSS 实现深蓝渐变、网格和点阵纹理。
- 项目图片由 PNG 转为 WebP，体积从约 8MB 降至约 365KB。
- 图片添加 `loading="lazy"`、`decoding="async"` 和低优先级加载。
- 滚动监听集中交给 GSAP ScrollTrigger 管理。
- 卡片光效通过 `requestAnimationFrame` 节流，避免频繁触发重绘。
- `npm run build` 已验证通过。

## 白屏问题记录

开发过程中尝试接入 React Bits 的 Grainient / WebGL 背景组件。接入后出现页面白屏，但本地 Vite 服务仍正常返回页面。

判断原因：

- 问题不是普通 CSS 冲突。
- 更可能是 WebGL2 / canvas 初始化在当前预览环境中不稳定。
- WebGL 初始化异常会中断 React 页面渲染，导致 DOM 没有正常挂载。

最终处理：

- 移除 WebGL 背景方案。
- 改为纯 CSS 深蓝渐变、细网格、点阵和纹理背景。
- 保留视觉质感，同时提高兼容性和稳定性。
